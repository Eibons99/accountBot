## 简介

**Accounting Bot** 是一款基于 Telegram 的智能记账机器人，支持个人和群组记账功能。

### 核心特性
- ✅ 支持个人记账和群组 AA 制记账
- ✅ 自动识别操作员和标记员角色
- ✅ 支持备注、分类统计
- ✅ Redis 缓存机制，防止数据丢失
- ✅ 支持 LongPolling 和 Webhook 两种运行模式

## 环境准备

### 1. 软件要求
- **JDK**: 1.8 或更高版本
- **Maven**: 3.6+
- **MySQL**: 5.7+ 或 8.0+
- **Redis**: 5.0+

### 2. 数据库初始化

#### MySQL 配置
```sql
-- 创建数据库
CREATE DATABASE accounting CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户（可选）
CREATE USER 'root'@'%' IDENTIFIED BY 'Kx9mP2vL7nQ4wRbZ';
GRANT ALL PRIVILEGES ON accounting.* TO 'root'@'%';
FLUSH PRIVILEGES;
```

#### Redis 配置
已提供 `redis.conf` 配置文件，启动命令：
```bash
# 使用配置文件启动 Redis
redis-server redis.conf

# 验证连接
redis-cli -h 82.40.37.166 -p 63179 -a Rx5mK8vN2pL6wQjT
```

### 3. Telegram Bot 申请
1. 在 Telegram 中搜索 `@BotFather`
2. 发送 `/newbot` 创建新机器人
3. 按提示设置机器人名称和用户名
4. 获取 Token（格式如：`8629295100:AAGm7k6BWbP7N1TcshL9NfMC7R5Qm3CdBUI`）

## 快速开始

### 1. 克隆项目
```bash
git clone <repository-url>
cd accounting-bot
```

### 2. 修改配置

编辑 `src/main/resources/application.yml`：

```yaml
# 服务器地址（支持本地调试）
my:
  server: 127.0.0.1  # 本地调试改为 127.0.0.1，生产环境改为实际 IP

spring:
  datasource:
    url: jdbc:mysql://${my.server}:13306/accounting?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: Kx9mP2vL7nQ4wRbZ
  
  redis:
    host: ${my.server}
    port: 63179
    password: Rx5mK8vN2pL6wQjT
    timeout: 10000ms  # 连接超时时间（毫秒）

bot:
  token: <你的 Bot Token>
  mode: longpolling  # 或 webhook
```

### 3. 编译打包
```bash
mvn clean package -DskipTests
```

### 4. 启动应用

#### 方式一：直接运行 JAR
```bash
java -jar target/accounting-bot-0.0.1-SNAPSHOT.jar
```

#### 方式二：使用 Maven 启动
```bash
mvn spring-boot:run
```

#### 方式三：后台运行（生产环境）
```bash
nohup java -jar accounting-bot.jar > bot.log 2>&1 &
```

### 5. 验证启动
查看日志确认启动成功：
```
2024-01-01 12:00:00 [main] INFO  - 机器人已启动（LongPolling 模式）
```

## 配置说明

### 1. 数据库配置
```yaml
spring:
  datasource:
    url: jdbc:mysql://<host>:<port>/accounting
    username: <数据库用户名>
    password: <数据库密码>
```

### 2. Redis 配置
```yaml
spring:
  redis:
    host: <Redis 主机>
    port: <Redis 端口>
    password: <Redis 密码>
    database: 7  # 使用的数据库编号
    timeout: 10000ms  # 连接超时（建议不少于 5 秒）
    lettuce:
      pool:
        max-active: 8   # 最大连接数
        max-idle: 8     # 最大空闲连接
        min-idle: 0     # 最小空闲连接
```

### 3. Bot 配置
```yaml
bot:
  username: 超级记账机器人
  token: <从 BotFather 获取的 Token>
  
  # 运行模式
  mode: longpolling  # longpolling（长轮询）或 webhook（回调）
  
  # Webhook 配置（仅 webhook 模式需要）
  webhook-url: https://your-domain.com
  webhook-path: /webhook
  server-port: 8443
  
  # 记账配置
  exchange-rate: 1      # 汇率（群组记账转换为本位币）
  fee-rate: 0          # 手续费率
  day-cutoff-time: 00:00  # 日切时间（每日账单分割时间点）
```

### 4. 日志配置
```yaml
logging:
  level:
    com.bot.accounting: INFO  # 业务日志级别
    org.telegram: WARN        # Telegram SDK 日志级别
```

## 功能使用

### 基础指令

#### 1. 启动机器人
```
/start
```
机器人会回复欢迎信息和可用指令列表。

#### 2. 记账指令

##### 查看人员列表
```
/标记员列表      # 查看所有标记人员
/操作员列表      # 查看所有操作员
```

##### 个人记账（操作员）
```
100 早餐
```
格式：`金额 备注`

##### 群组记账（标记员）
```
@username 100 午餐
```
格式：`@成员 金额 备注`

##### 多笔连续记账
```
100 早餐
50 交通
@Alice 80 购物
```

#### 3. 查询指令

##### 查看今日统计
```
/today
```
显示今日收支汇总。

##### 查看历史统计
```
/history
```
按日期显示历史账单。

##### 查看成员账单
```
/member @Alice
```
查看指定成员的账单详情。

##### 查看人员权限
```
/标记员列表
/操作员列表
```
查看当前群组的标记员和操作员名单。

### 角色说明

#### 操作员（Operator）
- 可以直接发送「金额 + 备注」进行记账
- 通常是群主或管理员

#### 标记员（Tagger）
- 需要使用 `@成员` 方式记账
- 系统会自动记录标记关系

### 高级功能

#### 1. 账单更正
如果发现记错账，可以：
- 删除错误消息，机器人会自动同步删除
- 联系管理员在后台修正

#### 2. 数据导出
```
/export 2024-01
```
导出指定月份的账单数据（CSV 格式）。

#### 3. AA 制结算
群组记账会自动计算每个成员的应分摊金额。

## 常见问题

### 1. Redis 连接超时
**错误信息**：
```
io.lettuce.core.RedisConnectionException: Unable to connect to <host>:<port>
```

**解决方案**：
1. 检查 Redis 服务是否启动：`redis-cli ping`
2. 检查防火墙是否开放端口
3. 增加超时时间：`timeout: 10000ms`
4. 验证密码是否正确：`redis-cli -a <password> ping`

### 2. Bot 无法启动（409 Conflict）
**错误信息**：
```
Conflict: can't use getUpdates method while webhook is active
```

**原因**：Telegram 服务器仍认为机器人在 Webhook 模式

**解决方案**：
```java
// 临时方案：先删除 webhook
SetWebhook setWebhook = new SetWebhook();
execute(setWebhook);

// 或者等待 5-10 分钟后重试
```

### 3. 数据库连接失败
**检查项**：
1. MySQL 服务是否运行
2. 端口是否正确（默认 3306 或自定义 13306）
3. 用户名密码是否正确
4. 数据库是否已创建

### 4. 消息无响应
**可能原因**：
- Bot 未成功启动
- Telegram API 被墙（需要代理）
- Bot Token 无效

**排查步骤**：
```bash
# 测试 Telegram API 连通性
curl https://api.telegram.org/bot<TOKEN>/getMe

# 查看应用日志
tail -f bot.log
```

### 5. Redis 持久化问题
如果遇到数据丢失：
1. 检查 `redis.conf` 中 AOF 是否开启：`appendonly yes`
2. 检查磁盘空间是否充足
3. 查看 Redis 日志：`tail -f /logs/redis.log`

## 运维管理

### 1. 日志管理

#### 查看实时日志
```bash
tail -f bot.log
```

#### 按关键字搜索
```bash
grep "ERROR" bot.log | tail -50
```

#### 日志轮转（logback.xml）
配置日志文件大小和保留天数，避免磁盘占满。

### 2. 监控指标

#### 应用健康检查
```bash
# 检查进程是否存在
ps aux | grep accounting-bot

# 检查端口监听
netstat -tlnp | grep <port>
```

#### Redis 监控
```bash
# 连接数
redis-cli -a <password> info clients

# 内存使用
redis-cli -a <password> info memory

# 持久化状态
redis-cli -a <password> info persistence
```

#### 数据库监控
```sql
-- 查看连接数
SHOW STATUS LIKE 'Threads_connected';

-- 查看慢查询
SHOW VARIABLES LIKE 'slow_query_log';
```

### 3. 备份策略

#### 数据库备份
```bash
# 每日备份
mysqldump -h <host> -u root -p accounting > accounting_$(date +%Y%m%d).sql

# 定期清理旧备份
find /backup -name "accounting_*.sql" -mtime +30 -delete
```

#### Redis 备份
Redis 会自动保存 RDB 和 AOF 文件，位置：
- RDB: `/data/dump.rdb`
- AOF: `/data/appendonly.aof`

建议定期复制到其他服务器。

### 4. 故障恢复

#### Bot 异常退出
```bash
# 重启应用
java -jar accounting-bot.jar > bot.log 2>&1 &

# 验证启动
tail -f bot.log
```

#### Redis 数据恢复
```bash
# 停止 Redis
redis-cli shutdown

# 复制备份文件
cp /backup/dump.rdb /data/

# 重启 Redis
redis-server redis.conf
```

### 5. 性能优化

#### JVM 调优
```bash
# 根据服务器配置调整堆内存
java -Xms512m -Xmx2g -jar accounting-bot.jar
```

#### Redis 连接池
如果并发量大，可增加连接池大小：
```yaml
lettuce:
  pool:
    max-active: 16
    max-idle: 8
```
